Javascript의 모듈 시스템에는 크게 아래의 2가지가 있음(물론 더 있긴 하지만 논외로 함)
- CommonJS(cjs)
	- node.js 제안
	- ems가 없을 당시 제안했었음
	- 로컬 파일을 동기적으로 가져옴
	-  값의 복사로 사용
- ECMAScript System(esm)
	- ES6에서 표준으로 제정
	- import, export 사용
	- 비동기적으로 로딩하여 모듈 로딩이 코드 실행을 막지는 않는다
	- 정적 분석을 할 수 있어서 엔진 실행전 모듈의 의존성 그래프를 분석하여 코드 최적화가 가능하다
	- 최신 브라우저와 node.js 13.2.0 이상에서 기본적으로 지원한다

Q. 왜 cjs의 방식이 제안된 것이였는가?
A. 기본적으로 javascript는 `<script>`를 사용하게 되면 전체가 로드되어 전역 공간의 스코프를 가지게 된다. 이러한 문제를 해결하고자 cjs는 모듈 래핑을 통하여서 코드를 모듈화했다. 즉, node.js가 파일로 나눠진 것들을 모듈 래핑하여 스코프를 격리했다(함수 안으로 감싸짐) 
함수로 래핑하여 표현식으로 만든다음 require로 경로가 호출되면 require.cache에서 확인하여 없으면 레코드를 추가하고 즉시 실행 함수(IIFE, Immediately Invoke Function Expression) 방법으로 함수를 정의하자마자 바로 실행시켜 전역 스코프 오염을 방지하고 모두 함수 안쪽으로 감싼다 

Q. ESM은 어떻게 모듈 시스템을 구현하였는가?
A. 1) 정적 분석을 통해 import와 export 키워드를 분석하여 모듈 의존성 그래프를 완성한다
2) 의존성 그래프를 바탕으로 각 모듈의 정보를 담은 모듈 레코드를 생성(모듈맵)하고 모듈 간의 연결을 설정한다
3) import 해온 변수가 원본 모듈의 변수에 대한 메모리상의 실제 연결을 유지한다(CJS는 값을 복사해옴)
