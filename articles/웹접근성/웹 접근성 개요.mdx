# 스펙간의 상관 관계
- WCAG(Web Contents Accessibility Guide)
- ARIA(Accessible Rich Internet Application)
웹 접근성하면 위의 2개 큰 문서가 있음
WCAG는 무엇을 충족시켜야 하는지 말하고 있고 ARIA는 WCAG에 적혀있는 것을 충족 시키는 방법을 기술해놓은 문서임

# WCAG
WCAG에는 크게 4가지의 영역이 있음
- Perceivable(인지 가능성)
	- (일단 컨텐츠가 뭔지 알아야)
- Operable(운영 가능성)
	- (인지 했으면 움직여야 하니깐)
- Understandable(이해 가능성)
	- (인지하고 운영할 수 있는데, 일단 뭔지 가늠이 되어야 하니깐)
- Robust(견고성)
	- (위의 것들을 다 했는데 내가 사용하는 에이전트, 장치에서도 실행해야 하니깐)

## Perceivable(인지 가능성)
- 텍스트 대안
- 시각기반 미디어
- 적응성
	- 정보/구조 손실 X, 다른 방식으로도 보여줄 수 있어야(레이아웃)
- 구별 가능성
	- 좀 보여야지
## Operable(운영 가능성)
- 키보드
	- 일반적인 작동 방식
- 충분한 시간
	- 직접 움직여서 해야하기 때문에 시간이 필요
- 신체 반응
	- 놀라게 하지 마라
- 내비게이션
	- 안그래도 불편한데 이거라도 있어야
- 다양한 입력방식
	- 입력은 키보드만으로 하는 것이 아니다
## Understanable(이해 가능성)
정보, 인터페이스가 이해할 수 있어야
- 가독성
	- 잘 보여야 이해하지
- 예측 가능성
	- 예상한대로 동작해야지(맨날 하던거 비슷하게 하거라)
- 입력 지원
	- 입력할 때, 다양하게 입력할 수 있으니 좀 보정 좀 알아서 잘해달라
## Robust(견고성)
콘텐츠는 어떤 에이전트간 호환성을 극대화 시켜야
- 파싱(html 관련 내용인데 현재는 폐기)
- 이름, 역할, 값
	- 그나마 추상화한 값인듯
- 상태 메시지
	- 상태도 남겨야 뭔지 알지
