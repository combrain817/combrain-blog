
- HTML5 초기에 Document outline algorithm이라는 개념이 있었음
	- 문서의 구조적인 개요를 생성하는 알고리즘임
	- 태그에서 레벨을 수동으로 관리하지 않고 섹셔닝 컨텐츠 요소(section, article, nav, asice)의 중첩 구조에 따라 헤딩의 계층을 자동으로 결정하는 것
	- 하지만 주요 웹 브라우저 및 스크린 리더에서 해당 기술을 구현된 적이 없음
	- 따라서 논리적 순서에 맞춰 `<h1>~<h6>`까지의 헤딩 레벨을 직접적으로 명시적으로 정의해야 함

- 왜 각 섹션별로 h1을 작성하면 안되는가? 문서 전체에서 왜 h1이 하나가 되도록 구조를 권장하는가?
	- 이유는 다음과 같다
		- 명확한 페이지 정체성
			- 사용자와 검색엔진 모두에게 "이 페이지는 무엇에 관한 것인가?" 라는 질문을 했을 때 대답을 할 수 있는 영역이다
		- 검색 엔진 최적화
			- 검색 엔진이 페이지의 핵심 컨텐츠를 식별하는 매우 중요한 신호로 사용한다
			- h1이 여러 개일 경우 페이지의 핵심 주제에 대한 가중치가 분산되어 SEO에 불리하게 작용될 수 있다
		- 웹 접근성
			- h1이 하나만 존재하고 아래에 순차적으로 잘 적용되어 있으면 탐색하기 편한데 h1이 여러개이면 페이지를 탐색하기 어렵다
