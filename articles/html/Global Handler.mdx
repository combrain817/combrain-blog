### 마우스 이벤트 (Mouse Events)

- `onclick`: 사용자가 요소를 클릭했을 때 발생한다. (주로 마우스 왼쪽 버튼)
- `onauxclick`: 사용자가 요소의 보조 버튼(일반적으로 마우스 휠 버튼 또는 오른쪽 버튼)을 클릭했을 때 발생한다.
- `oncontextmenu`: 사용자가 요소의 컨텍스트 메뉴를 열려고 시도할 때(주로 마우스 오른쪽 클릭) 발생한다. `return false;`를 통해 기본 메뉴가 나타나는 것을 막을 수 있다.
- `ondblclick`: 사용자가 요소를 더블 클릭했을 때 발생한다.
- `onmousedown`: 사용자가 요소 위에서 마우스 버튼을 눌렀을 때 발생한다.
- `onmouseup`: 사용자가 요소 위에서 눌렀던 마우스 버튼을 놓았을 때 발생한다.
- `onmouseenter`: 마우스 포인터가 요소의 경계 안으로 들어왔을 때 발생한다. 자식 요소를 드나드는 것에는 영향을 받지 않는다.
- `onmouseleave`: 마우스 포인터가 요소의 경계 밖으로 나갔을 때 발생한다. `onmouseenter`와 마찬가지로 버블링되지 않는다.
- `onmousemove`: 마우스 포인터가 요소 위에서 움직일 때마다 발생한다.
- `onmouseover`: 마우스 포인터가 요소 또는 그 자식 요소 위로 올라왔을 때 발생한다. (버블링 발생)
- `onmouseout`: 마우스 포인터가 요소 또는 그 자식 요소 밖으로 나갔을 때 발생한다. (버블링 발생)
- `onwheel`: 사용자가 마우스 휠을 돌리거나 트랙패드를 스크롤할 때 발생한다.

### 키보드 이벤트 (Keyboard Events)

- `onkeydown`: 사용자가 키를 눌렀을 때 발생한다. 키를 누르고 있는 동안 연속적으로 발생할 수 있다.
- `onkeyup`: 사용자가 눌렀던 키를 놓았을 때 발생한다.
- `onkeypress`: `onkeydown` 이후에 발생하며, 문자를 생성하는 키(예: a, b, 1, 2)에 대해서만 작동한다. 현재는 사용이 권장되지 않으며(deprecated), `onbeforeinput`이나 `onkeydown` 사용이 권장된다.

### 포커스 이벤트 (Focus Events)

- `onfocus`: 요소가 포커스를 받았을 때 발생한다. (예: 탭 키로 선택되거나 마우스로 클릭했을 때)
- `onblur`: 요소가 포커스를 잃었을 때 발생한다.

> `*` **`onfocus`와 `onblur`의 특별한 동작**: `<body>` 요소에 사용될 경우, 이는 `window` 객체의 `focus`, `blur` 이벤트를 처리한다. 즉, 브라우저 창이나 탭 자체가 활성화되거나 비활성화될 때 동작한다.

### 폼 및 입력 이벤트 (Form & Input Events)

- `onchange`: `<input>`, `<select>`, `<textarea>` 요소의 값이 변경되고 포커스를 잃었을 때(committed) 발생한다.
- `oninput`: `<input>`, `<textarea>` 등 요소의 `value`가 사용자에 의해 변경될 때마다 실시간으로 발생한다.
- `onbeforeinput`: 요소의 값이 수정되기 직전에 발생한다. `oninput`과 달리 변경을 취소할 수 있다.
- `oninvalid`: 폼 제출 시 유효성 검사(예: `required`, `pattern`)를 통과하지 못한 요소에서 발생한다.
- `onreset`: 폼의 리셋 버튼(`<input type="reset">` 또는 `<button type="reset">`)을 클릭했을 때 `<form>` 요소에서 발생한다.
- `onsubmit`: 폼이 제출될 때 `<form>` 요소에서 발생한다. `return false;`로 폼 제출을 막을 수 있다.
- `onselect`: `<input>`이나 `<textarea>`에서 사용자가 텍스트를 선택했을 때 발생한다.
- `onformdata`: `FormData` 객체가 생성된 후, 폼 제출이 일어나기 전에 발생한다. 폼 데이터를 스크립트로 수정할 기회를 제공한다.

### 미디어 이벤트 (Media Events) - `<audio>`, `<video>` 등

- `oncanplay`: 브라우저가 재생을 시작할 수 있을 만큼 충분한 데이터를 다운로드했을 때 발생한다. 버퍼링 없이 끝까지 재생할 수 있는지는 보장하지 않는다.
- `oncanplaythrough`: 버퍼링 없이 미디어를 끝까지 재생할 수 있을 것으로 예상될 때 발생한다.
- `ondurationchange`: 미디어의 길이(duration) 정보가 변경되었을 때 발생한다.
- `onemptied`: 네트워크 오류 등으로 미디어 데이터가 비었을 때 발생한다.
- `onended`: 미디어 재생이 끝에 도달했을 때 발생한다.
- `onloadeddata`: 현재 재생 위치의 미디어 데이터를 로드했을 때 발생한다.
- `onloadedmetadata`: 미디어의 메타데이터(길이, 크기 등)를 성공적으로 로드했을 때 발생한다.
- `onloadstart`: 브라우저가 미디어 데이터 로드를 시작했을 때 발생한다.
- `onpause`: 미디어 재생이 일시 중지되었을 때 발생한다.
- `onplay`: 미디어 재생이 시작되었을 때 발생한다.
- `onplaying`: 일시 중지 또는 데이터 부족으로 인한 버퍼링 후 재생이 다시 시작될 때 발생한다.
- `onprogress`: 브라우저가 미디어 데이터를 다운로드하는 동안 주기적으로 발생한다.
- `onratechange`: 재생 속도(playback rate)가 변경되었을 때 발생한다.
- `onseeked`: 탐색(seeking) 작업이 완료되었을 때 발생한다.
- `onseeking`: 탐색 작업이 시작되었을 때 발생한다.
- `onsuspend`: 미디어 데이터 로드가 의도적으로 중단되었을 때(전체 다운로드 후 등) 발생한다.
- `onstalled`: 브라우저가 미디어 데이터를 가져오려고 시도했지만 데이터가 사용 불가능할 때 발생한다.
- `ontimeupdate`: 현재 재생 위치(currentTime)가 변경될 때마다 주기적으로 발생한다.
- `onvolumechange`: 볼륨이 변경되거나 음소거(muted) 상태가 변경되었을 때 발생한다.
- `onwaiting`: 데이터 부족으로 인해 다음 프레임을 사용할 수 없어 재생이 일시적으로 멈췄을 때(버퍼링) 발생한다.

### 드래그 앤 드롭 이벤트 (Drag & Drop Events)

- `ondrag`: 드래그 가능한 요소를 드래그하는 동안 발생한다.
- `ondragstart`: 사용자가 요소 드래그를 시작할 때 발생한다.
- `ondragend`: 드래그 작업이 끝났을 때(드롭 성공, 취소 등) 발생한다.
- `ondragenter`: 드래그 중인 항목이 유효한 드롭 대상(drop target) 요소 위로 들어왔을 때 발생한다.
- `ondragleave`: 드래그 중인 항목이 유효한 드롭 대상 요소 밖으로 나갔을 때 발생한다.
- `ondragover`: 드래그 중인 항목이 유효한 드롭 대상 요소 위에 머물러 있을 때 주기적으로 발생한다. 드롭을 허용하려면 이 이벤트의 기본 동작을 `preventDefault()`로 막아야 한다.
- `ondrop`: 드래그 중인 항목을 유효한 드롭 대상 위에 드롭했을 때 발생한다.

### 복사/붙여넣기 이벤트 (Clipboard Events)

- `oncopy`: 사용자가 요소의 콘텐츠를 복사했을 때 발생한다.
- `oncut`: 사용자가 요소의 콘텐츠를 잘라냈을 때 발생한다.
- `onpaste`: 사용자가 요소에 콘텐츠를 붙여넣었을 때 발생한다.

### 전역 및 기타 이벤트 (Global & Other Events)

- `onload`: 페이지의 모든 콘텐츠(이미지, 스크립트, CSS 등)가 완전히 로드된 후에 `window` 객체에서 발생한다.
- `onerror`: 리소스(이미지, 스크립트) 로드에 실패했거나 JavaScript 코드 실행 중 오류가 발생했을 때 발생한다.
- `onresize`: 문서 뷰(브라우저 창)의 크기가 조절되었을 때 발생한다.
- `onscroll`: 문서 뷰나 특정 요소가 스크롤될 때 발생한다.
- `onscrollend`: 사용자의 스크롤이 끝났을 때 발생한다.

> `*` **`onload`, `onerror`, `onresize`, `onscroll`, `onscrollend`의 특별한 동작**: 이 핸들러들을 `<body>` 요소에 지정하면, 실제로는 `window` 객체에 대한 이벤트 핸들러로 동작한다. 예를 들어 `<body>`의 `onresize`는 브라우저 창 크기 조절 이벤트를 감지한다.

- `oncancel`: `<dialog>` 요소가 `Esc` 키로 닫힐 때 발생한다.
- `onclose`: `<dialog>` 요소가 닫힐 때 발생한다.
- `onbeforematch`: (실험적) `hidden="until-found"` 요소에 대한 브라우저의 '페이지에서 찾기' 기능이 일치하는 항목을 찾았을 때 발생한다.
- `onbeforetoggle`: `popover`나 `details` 요소의 상태가 열림/닫힘으로 전환되기 직전에 발생한다.
- `ontoggle`: `popover`나 `details` 요소의 상태가 전환된 후에 발생한다.
- `onsecuritypolicyviolation`: 콘텐츠 보안 정책(Content Security Policy) 위반이 발생했을 때 발생한다.
- `onslotchange`: 웹 컴포넌트의 `<slot>`에 할당된 노드가 변경되었을 때 발생한다.
- `oncontextlost` / `oncontextrestored`: WebGL 컨텍스트가 손실되거나 복원될 때 `<canvas>` 요소에서 발생한다.
- `oncuechange`: `<track>` 요소의 텍스트 트랙 큐(cue)가 변경될 때 발생한다.
- `oncommand`: (사용이 저조함) 사용자가 command를 호출했을 때 발생한다.
