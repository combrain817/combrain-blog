## 무엇?
html 태그 아래에 위치하는 태그로 문서의 실제 내용이 들어간다.
문서에 하나만 있어야 한다

## 핸들러
- Global handler 모두 포함
### 문서 인쇄 이벤트

- **`onbeforeprint`**: 사용자가 페이지 인쇄를 요청하여 인쇄 대화상자가 나타나기 직전에 발생한다. 인쇄용 스타일을 동적으로 적용할 때 사용될 수 있다.
- **`onafterprint`**: 인쇄 대화상자가 닫히거나 인쇄 작업이 시작된 후에 발생한다.

---

### 문서/윈도우 생명주기(Lifecycle) 이벤트

- **`onbeforeunload`**: 문서나 창이 닫히기 직전에 발생한다. 사용자가 페이지를 떠나는 것을 막고 확인 메시지를 표시하는 데 사용될 수 있으나, 브라우저 보안 정책에 따라 표시되는 메시지는 제한적이다.
- **`onunload`**: 페이지가 언로드될 때 발생한다. 리소스 해제 등의 목적으로 사용되었으나, 최신 브라우저의 뒤로/앞으로 캐시(bfcache) 기능을 방해할 수 있어 `pagehide` 사용이 권장된다.
- **`onpagehide`**: 사용자가 페이지를 떠날 때 발생한다. `unload` 이벤트와 달리 뒤로/앞으로 캐시에 페이지가 저장될 때도 발생한다.
- **`onpageshow`**: 페이지가 사용자에게 표시될 때 발생한다. 페이지가 처음 로드될 때나 뒤로/앞으로 캐시에서 복원될 때 모두 발생한다. `event.persisted` 프로퍼티로 캐시에서 복원되었는지 확인할 수 있다.
- **`onpageswap`**: 문서가 뒤로/앞으로 캐시(bfcache)에 의해 새 문서로 교체될 때 발생한다. 이 시점부터 문서는 사용자에게 보이지 않게 된다.
- **`onpagereveal`**: 문서가 뒤로/앞으로 캐시에서 복원되어 사용자에게 다시 표시될 때 발생한다.

---

### 브라우저 히스토리 및 탐색 이벤트

- **`onpopstate`**: 사용자의 브라우저 세션 히스토리가 변경될 때(예: '뒤로', '앞으로' 버튼 클릭) 발생한다. `history.pushState()`나 `history.replaceState()` 호출만으로는 발생하지 않는다.
- **`onhashchange`**: URL의 해시(fragment), 즉 `#` 뒷부분이 변경되었을 때 발생한다. Single Page Application(SPA)에서 라우팅에 주로 활용된다.

---

### 네트워크 상태 이벤트

- **`ononline`**: 브라우저가 네트워크 연결을 감지했을 때 발생한다.
- **`onoffline`**: 브라우저가 네트워크 연결이 끊겼음을 감지했을 때 발생한다.

---

### Promise 비동기 처리 이벤트

- **`onunhandledrejection`**: JavaScript `Promise`가 reject되었지만, 이를 처리할 `.catch()` 핸들러가 없을 때 발생한다. 전역 에러 로깅에 유용하다.
- **`onrejectionhandled`**: `Promise`가 reject된 후, 뒤늦게라도 reject를 처리하는 핸들러가 추가되었을 때 발생한다.

---

### 기타 이벤트

- **`onstorage`**: `localStorage` 또는 `sessionStorage`의 내용이 **다른** 문서(예: 다른 탭)에서 변경되었을 때, 동일한 오리진(origin)을 가진 문서들에서 발생한다. 탭 간 상태 동기화에 사용된다.
- **`onmessage`**: 다른 `window`, `iframe`, 또는 `Worker`로부터 `postMessage()` API를 통해 메시지를 수신했을 때 발생한다.
- **`onmessageerror`**: `postMessage()`로 받은 메시지를 역직렬화(deserialize)할 수 없을 때 발생한다.
- **`onlanguagechange`**: 사용자의 선호 언어 설정(브라우저 또는 OS)이 변경되었을 때 발생한다.
